# 🧠 Engram-PEFT 短期记忆更新规范 (Short-Term Memory Update)

本技能用于**会话级短期记忆维护**，严格区分「短期临时上下文」和「长期架构沉淀」，避免记忆膨胀、上下文丢失。

## 1. 核心定义
- **短期记忆**：当前会话专属，仅对本次任务有效，会话结束可清理/归档
  - 临时TODO、当前会话踩坑、未闭环问题、即时排错记录
- **长期记忆**：跨会话有效，永久沉淀
  - 架构决策、人类决策、通用Bug坑点、版本兼容问题

## 2. 短期记忆强制写入步骤
1. **读取现有记忆**
   先读取 `.agent_memory.md` 全文，避免重复记录、覆盖关键信息
2. **分类写入**
   - 临时任务 → 写入 `Active TODOs`（短期）
   - 本次会话踩坑 → 写入 `Gotchas & Context` 并标注「短期」
   - 熔断排错 → 写入 `Gotchas & Context` + 补充临时TODO
3. **格式校验**
   禁止自由文本，必须沿用AGENTS.md规定的Markdown Schema
4. **冗余检查**
   同一问题/任务不重复记录，合并同类项

## 3. 会话收尾：短期记忆归档与修剪
每次会话结束**必须执行**：
1. **清理已完成短期TODO**，直接删除，不保留冗余
2. **有价值的短期踩坑** → 移入长期 `Gotchas`，移除「短期」标记
3. **无价值临时上下文** → 直接删除
4. 检查文件行数，超过200行则精简压缩

## 4. 死循环熔断后的记忆写入规范
熔断排错总结必须**完整写入短期记忆**，格式固定：
```markdown
## ⚠️ Gotchas & Context
- 【熔断短期】3次重试修复XXBug失败，报错：XXX；尝试方案：1.XX 2.XX 3.XX；猜想：XXX
```

## 5. 模板初始化规范
若 `.agent_memory.md` 不存在：
1. 直接复制 `.agent_memory.template.md` 完整内容
2. 清空短期TODO，仅保留长期架构/人类决策基线
3. 不修改模板结构，仅追加当前会话信息

## 6. 禁止行为
- 禁止直接删除长期记忆（Architecture Decisions/Human Decisions）
- 禁止在短期记忆区写入跨会话通用规则
- 禁止使用非规范格式、随意换行/无列表
- 禁止记忆文件超过300行（强制修剪）
