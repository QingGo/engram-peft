# 📚 文档同步规范

代码与文档的一致性是项目可用性的关键。以下情况必须同步更新对应的文档：

## 1. API 文档更新
- 修改了任何公开函数或类的参数、返回值或行为 → 更新 `docs/api/`
- 新增了公开API → 在 `docs/api/` 中添加对应的文档
- 废弃了某个API → 添加弃用警告并在文档中说明替代方案

## 2. 教程与示例更新
- 新增了训练模式或功能 → 更新 `docs/tutorials/`
- 变更了使用方法 → 同步更新 `examples/` 中的示例代码
- 修复了示例中的错误 → 同时更新文档中的对应部分

## 3. README 更新
- 项目的核心功能或特性发生变化 → 更新 README.md 和 README_zh.md
- 性能基准更新 → 更新 README 中的性能对比表格
- 安装步骤变更 → 更新 README 中的 Quick Start 部分

## 4. 文档质量要求
- 所有代码示例必须可直接运行
- 所有参数必须有明确的说明和默认值
- 中英文文档必须保持同步
- 避免使用过于技术化的术语，保持文档的可读性
